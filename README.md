# LADAL Jupyter Notebook Tools

Interactive, browser-based text analysis tools built on Jupyter notebooks with R.
No installation required — just click a launch link and start working with your own texts.

Developed and maintained by the [Language Technology and Data Analysis Laboratory (LADAL)](https://ladal.edu.au) at the University of Queensland.

---

## How to Use

1. Click a launch link in the table below.
2. Wait for the environment to load (first launch: 10–20 minutes; subsequent launches: under a minute).
3. Once JupyterLab opens, the notebook loads automatically.
4. **Upload your files** by dragging them into the relevant folder in the file browser panel on the left.
5. **Edit the configuration cell** (Step 3 in each notebook) if needed.
6. Click **Kernel → Restart & Run All** to run the analysis.
7. **Download your results** from the **MyOutput** folder (right-click → Download).

## Uploading Your Files

Each tool reads from a specific folder in the file browser on the left side of JupyterLab:

| Tool | Upload folder | File type |
|---|---|---|
| All text tools | `MyTexts` | `.txt` or `.TXT` |
| Keyword Finder (target texts) | `Target` | `.txt` or `.TXT` |
| Keyword Finder (reference texts) | `Reference` | `.txt` or `.TXT` |
| Network Visualiser | `MyTables` | `.xlsx` |

Results are always saved to the `MyOutput` folder. Right-click any file there and choose **Download**.

## Repository Structure

```
jupyter/
├── notebooks/
│   ├── concordance_explorer.ipynb
│   ├── collocation_analyser.ipynb
│   ├── keyword_finder.ipynb
│   ├── pos_tagger.ipynb
│   ├── text_cleaner.ipynb
│   ├── network_visualiser.ipynb
│   ├── topic_explorer.ipynb
│   ├── sentiment_explorer.ipynb
│   ├── MyTexts/        ← upload plain-text files here
│   ├── Target/         ← upload target corpus here (Keyword Finder)
│   ├── Reference/      ← upload reference corpus here (Keyword Finder)
│   ├── MyTables/       ← upload edge-list Excel file here (Network Visualiser)
│   └── MyOutput/       ← download results from here
└── .Rprofile           ← disables renv auto-activation
```

## Environment

R packages are managed separately in [SLCLADAL/jupyter-env](https://github.com/SLCLADAL/jupyter-env).
Keeping content and environment in separate repositories means notebook updates do not trigger a full package rebuild on Binder.

## Generating New Launch Links

Use the [nbgitpuller link generator](https://hub.jupyter.org/nbgitpuller/link.html) with:

- **BinderHub URL**: one of the four platforms listed above
- **Environment repo**: `https://github.com/SLCLADAL/jupyter-env` (branch: `main`)
- **Content repo**: `https://github.com/SLCLADAL/jupyter` (branch: `main`)
- **File to open**: `notebooks/<filename>.ipynb`
- **Application**: JupyterLab
